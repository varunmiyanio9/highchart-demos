Bucket Selection — Multi Chart Types

Selection is shared across all charts. Click, Ctrl+Click, or drag to select buckets.

Selected Buckets Detail

Click to select entire category. Ctrl+Click to multi-select. Drag to select range. Keyboard: Tab into chart → Arrow keys → Space to toggle. Escape to clear.

Custom: Shared selection state (Set) across 3 chart types — Highcharts has no built-in cross-chart selection sync. syncAllCharts() propagates selection, plotBands highlight selected categories, and Shift+Arrow range select is implemented via global keydown tracking.

Selected Categories Detail

Click buttons to select categories. Ctrl+Click for multi-select. Shift+Click for range. Keyboard: Tab into buttons → Arrow keys to move → Space to toggle. Escape to clear.

Custom: External DOM buttons as category selectors — Highcharts has NO native axis-label click/select event. HTML toolbar with roving tabindex pattern provides full keyboard accessibility without conflicting with Highcharts' internal focus management.

Selected Categories Detail

Click x-axis labels to select. Ctrl+Click for multi-select. Click chart background to clear.

Custom: Clickable x-axis labels via useHTML + event delegation — Highcharts provides no label click event. Category index resolved via textContent (data-* attrs stripped by Highcharts HTML sanitization). PlotBands and label styles re-applied in redraw callback since plotBand changes destroy/recreate label DOM.

Combined Selection (SVG) — Bidirectional Label + Grid

Drag on chart background for range select. Click x-axis labels. Ctrl+Click for multi-select. Keyboard: Tab → Arrow → Space.

Selected Categories Detail

Drag to select range on grid. Click labels for single select. Ctrl+Click for multi. Shift+Arrow for range. Escape to clear.

Custom: Pure SVG label click (no useHTML) via tick.label.on('click') — undocumented but stable API. Background-only mouse select translates pixel→category index. Hover via .on('mouseover')/.css() since SVG has no CSS :hover. All handlers re-attached on every redraw because plotBands destroy/recreate tick elements. Font size/weight toolbar updates all 3 charts dynamically. SVG filter for label background highlight.

Forecast Charts — Historical vs Predicted

Each chart shows historical data (solid) and forecast data (dotted/translucent) with confidence bands. Orange line marks the forecast start.

Custom: Data transformation helpers only (padRight, padLeft, buildAreaRange, buildErrorBars, hexToRgba) — reshapes data into formats Highcharts expects. All visuals (plotLines, arearange, errorbar, dashStyle, linkedTo, stacking) are native Highcharts config. Charts 6–7 use native plotLine events for hover tooltip. Charts 8a–8b use chart.renderer to draw custom hoverable lines with callout labels — Highcharts plotLines don't support custom tooltip content on hover.

Custom: Minimal — hover-dim via series.setState('inactive') attached to Highcharts-rendered legend DOM elements in chart.events.render callback. Dummy series (data:[]) used as group headers and dividers in the native legend. labelFormatter returns custom HTML for styled swatches/icons. All rendering is Highcharts-managed.

Custom: None — Pure Highcharts config. 5 series types (column, stacked column, line, area) in one chart using dual y-axes, stack IDs for independent stacking groups, and grouping for side-by-side columns. No custom JS logic.

Custom: Entire legend is external HTML — Highcharts' built-in legend cannot render grouped headers with member swatches and "|" dividers. Hover-dim via series.group.attr({opacity}) (internal SVG element, not public API). Click-toggle wires to series.show()/hide(). Visual feedback via .msl-off CSS class.

Context Menu — Built-in & Custom Right-Click

Chart 1: Highcharts built-in exporting menu customized with type switching options.
Chart 2: Right-click for a custom nested context menu (industry standard).

Chart 1 (hamburger menu) — 100% Highcharts config, zero custom code. The library renders and manages everything.

Chart 2 (right-click nested menu) — Fully custom implementation. Highcharts provides NO right-click menu, NO nested submenus, NO highlight locking, and NO per-series targeting. Only chart.hoverSeries, series.setState(), series.update(), chart.exportChart(), fullscreen, and print are Highcharts API calls.
